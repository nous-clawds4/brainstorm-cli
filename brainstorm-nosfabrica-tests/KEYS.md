# Test Account Key Registry

This file documents the nostr keys available for the Brainstorm NosFabrica
test suite. **No secrets are stored in this repository.**

## Where Keys Live

All nsecs are stored as environment variables in a local file:

```
~/.config/nosfabrica-tests/keys.env
```

To load all keys into your shell:

```bash
source ~/.config/nosfabrica-tests/keys.env
```

Scripts should source this file at startup to access any test account key.

## Available Accounts

| Env Variable | Description | Persistent? |
|---|---|---|
| `NOUS_NSEC` | Nous's main nostr identity | Yes |
| `STRESS_COORDINATOR_NSEC` | Stress Test 1 coordinator (follows all 100 test accounts) | Yes |
| `STRESS_ACCOUNT_001_NSEC` … `STRESS_ACCOUNT_100_NSEC` | Stress Test 1 test accounts | Yes |

For full pubkey details of stress test accounts, see `stress-test-1/accounts.json`
and `stress-test-1/coordinator.json` (generated by `setup.sh`).

## Adding a New Test Account

1. Generate a keypair:
   ```bash
   nak key generate    # outputs nsec
   nak key public <nsec>  # outputs hex pubkey
   ```

2. Add an `export` line to `~/.config/nosfabrica-tests/keys.env`:
   ```bash
   export MYAGENT_NSEC="nsec1..."
   ```

3. Add a row to the table above (commit this file — it contains only public
   info and env variable names, never secrets).

4. If the account needs a kind:0 profile or kind:10002 relay list, publish
   those after creating the key.

## Naming Convention

- Environment variable: `<AGENT_NAME>_NSEC` (uppercase, underscores)
- Throwaway accounts: `THROWAWAY_<PURPOSE>_NSEC` — delete from keys.env after use
- Persistent accounts: use a clear name that describes the agent's role

## For Agents

When you're spun up to run a test:

1. Read this file to see what accounts are available
2. `source ~/.config/nosfabrica-tests/keys.env` to load the keys
3. Use `$NOUS_NSEC` (or whichever account the test needs) in your commands
4. If you need a new account, follow "Adding a New Test Account" above
