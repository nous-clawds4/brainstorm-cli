# Stress Test 1 — Burst Follow Propagation

Tests whether the NosFabrica Brainstorm pipeline can handle a burst of kind 3
follow events from multiple independent accounts within a short time window.

## Overview

**Setup** (run once): Create 100 test accounts, publish kind:0 profiles, have
the Stress Test Coordinator follow all 100, and verify the Brainstorm servers
registered them.

**Test** (run repeatedly): A window of 10 sender accounts each publish a
kind:3 following a window of 10 target accounts (~6s apart), then monitor
propagation across all 5 pipeline endpoints for each event.

## Nonce Rotation

Every run automatically uses a different sender/target combination, so each
publish event represents a genuine *change* in follow count. No reset is needed
between runs.

### How it works

State is tracked in `state.json`:

```json
{"nonce": 0}
```

The nonce determines sender and target ranges:

| Nonce | Senders | Targets | Notes |
|-------|---------|---------|-------|
| 0 | 1–10 | 11–20 | First run |
| 1 | 1–10 | 12–21 | Targets slide +1 |
| 2 | 1–10 | 13–22 | Targets slide +1 |
| … | 1–10 | … | |
| 80 | 1–10 | 91–100 | Last target slide for senders 1–10 |
| 81 | 2–11 | 12–21 | Senders shift +1, targets reset |
| 82 | 2–11 | 13–22 | Targets slide again |
| … | … | … | |
| 3320 | 81–90 | 91–100 | Last possible combination |

- **81 target positions** per sender position (target floor = sender_end + 1)
- **81 sender positions** (senders 1–10 through 81–90)
- **3,321 total unique combinations** before exhaustion
- Sender and target ranges **never overlap**

### Why this works

Because each run publishes a *different* set of followed accounts, the follow
count always changes. The test detects increment from baseline → no false
passes from stale data, and no need to reset between runs.

### Exhaustion

After 3,321 runs, the nonce space is exhausted. The script will print an
error and exit. Run `./reset.sh --all` to clear all follow lists and reset
the nonce to 0.

### Manual override

You can bypass the nonce system with explicit ranges:

```bash
./run-test.sh --senders 1-10 --targets 51-60
```

Manual overrides do **not** read or update the nonce.

## Accounts

The coordinator's metadata is in `coordinator.json`.

## Setup

```bash
cd brainstorm-nosfabrica-tests/stress-test-1
./setup.sh
```

Setup performs these steps:

1. Generates 100 keypairs + 1 coordinator keypair (if not already in keys.env)
2. Publishes kind:0 profiles for all 101 accounts to public relays
3. Publishes a kind:3 from the coordinator following all 100 test accounts
4. Waits for propagation and verifies all follows registered on both
   Brainstorm servers
5. Saves `accounts.json` and `coordinator.json`

Setup is **idempotent** — if keys already exist in keys.env, it reuses them.
If accounts.json already exists with valid pubkeys, it skips key generation.

## Running the Test

```bash
./run-test.sh                   # auto-derive ranges from nonce (recommended)
./run-test.sh --senders 1-10 --targets 51-60  # manual override (skips nonce)
```

### What Happens

1. **Nonce lookup:** Read `state.json`, derive sender/target ranges
2. **Baseline:** Query follow counts for each sender across all 5 endpoints
3. **Burst publish:** Each sender publishes a kind:3 following all target
   accounts. Events are spaced ~6 seconds apart.
4. **Monitor:** Query all 5 endpoints for all senders every 60 seconds.
   Continue until all have propagated or 1 hour elapses.
5. **Results:** Save structured JSON to `results/`
6. **Nonce increment:** Update `state.json` for the next run

### Timing Constants

| Constant | Value | Description |
|----------|-------|-------------|
| `BURST_INTERVAL` | 6s | Delay between each sender's publish |
| `INITIAL_WAIT` | 30s | Wait after last publish before first check |
| `RETRY_INTERVAL` | 60s | Wait between monitoring rounds |
| `MAX_ELAPSED` | 3600s | Maximum total time before giving up |

## Cleanup / Reset

With nonce rotation, you rarely need to reset. Use only when:
- All 3,321 nonce combinations are exhausted
- You want to manually clean up follow lists

```bash
./reset.sh                 # reset accounts 1-10 + nonce → 0
./reset.sh --senders 1-20  # reset a custom range + nonce → 0
./reset.sh --all           # reset ALL 100 accounts + nonce → 0
```

This publishes an empty kind:3 (no p-tags) for each specified account and
resets the nonce in `state.json` to 0.
